# Installers: Windows (MSI/NSIS), macOS (DMG + Notarization), Linux (AppImage/deb/Flatpak) (#57) — Design Proposal

> Status: design scaffold (not an implementation).

## Summary

No installer exists for any platform. Windows users must extract a zip (no Start Menu
shortcut, no Add/Remove Programs entry). macOS has no DMG. Linux has no `.deb`,
AppImage, or Flatpak. File association (`.photonic` extension) is implemented in Rust for
Windows (`crates/photonic-app/src/main.rs:581`, via `register_file_association()`) but
has no equivalent for macOS or Linux, and requires the installer to set up the registry
entry on Windows (the current runtime registration at line 92 requires elevation or
pre-existing registry state). This issue is downstream of issue #56 (release pipeline);
it adds installer packaging on top of the bare binary artifacts.

## Scope

**In**
- **Windows**: WiX v4 MSI (preferred) or NSIS installer; Start Menu shortcut; Add/Remove
  Programs entry; `.photonic` file association via registry; silent install mode.
- **macOS**: signed/notarized `.app` bundle + `.dmg` with drag-to-Applications; `Info.plist`
  document type and UTI for `.photonic`; Gatekeeper-compatible.
- **Linux**: AppImage (widest compat) + `.deb` (Debian/Ubuntu); `.desktop` entry; MIME
  type registration via `shared-mime-info`; optional Flatpak manifest.
- Hook each installer into the release pipeline from issue #56.

**Out**
- Homebrew cask (community-maintainable later).
- Chocolatey / winget package submission (requires public release first).
- Snap package.
- Auto-update mechanism (separate issue).

## Proposed Approach

### Windows — WiX v4 MSI

`cargo-wix` integrates WiX v4 with `cargo build`. Add to workspace:

```toml
# Cargo.toml [workspace.metadata.wix] or a separate wix.toml
[package.metadata.wix]
upgrade-guid = "<generate-uuid>"
path-guid = "<generate-uuid>"
```

Run `cargo wix init` to generate `wix/main.wxs`. Customise:
- `<Shortcut>` in `ProgramMenuFolder`.
- `<ProgId>` + `<Extension>` for `.photonic` association (replaces the runtime
  `register_file_association()` call at `main.rs:581`; the installer sets the registry
  keys as SYSTEM, removing the need for per-user runtime registration).
- `<Icon>` pointing to `logo_improved.svg` converted to `.ico`.

In the release workflow (issue #56):

```yaml
- name: Build MSI
  if: runner.os == 'Windows'
  run: |
    cargo install cargo-wix --locked
    cargo wix --nocapture
```

Sign the MSI with the same `signtool.exe` call used for the `.exe` (Authenticode).

### macOS — .app bundle + DMG

Use **`cargo-bundle`** to produce a `.app`:

```toml
# Cargo.toml [package.metadata.bundle]
[package.metadata.bundle]
name = "Photonic"
identifier = "dev.photonic.app"
icon = ["assets/icons/icon.icns"]
document_types = [
  { name = "Photonic Document", extensions = ["photonic"],
    role = "Editor", icon = "assets/icons/doc.icns" }
]
```

Wrap the bundle in a DMG with `create-dmg` (Node.js CLI or brew formula — available on
macOS runners):

```yaml
- name: Create DMG
  if: runner.os == 'macOS'
  run: |
    npm install -g create-dmg
    create-dmg 'target/release/bundle/osx/Photonic.app' dist/ \
      --overwrite --dmg-title "Photonic Installer"
```

Codesign the `.app` bundle (not just the binary) and notarize the DMG, using the same
`notarytool` flow from issue #56. The `Info.plist` generated by `cargo-bundle` covers
the `CFBundleDocumentTypes` / `UTExportedTypeDeclarations` needed for macOS document
association.

### Linux — AppImage + .deb

**AppImage** via `appimagetool`:

```yaml
- name: Build AppImage
  if: runner.os == 'Linux'
  run: |
    mkdir -p AppDir/usr/bin AppDir/usr/share/applications \
              AppDir/usr/share/icons/hicolor/256x256/apps
    cp target/release/photonic AppDir/usr/bin/
    cp assets/icons/photonic.png AppDir/usr/share/icons/hicolor/256x256/apps/
    cp packaging/photonic.desktop AppDir/usr/share/applications/
    # AppRun symlink and .desktop at root
    wget -q https://github.com/AppImage/AppImageKit/releases/latest/download/appimagetool-x86_64.AppImage
    chmod +x appimagetool-x86_64.AppImage
    ./appimagetool-x86_64.AppImage AppDir dist/Photonic-x86_64.AppImage
```

**.deb** via `cargo-deb`:

```toml
# Cargo.toml [package.metadata.deb]
[package.metadata.deb]
maintainer = "Photonic Contributors"
copyright = "2024-2025, Photonic Contributors"
license-file = ["LICENSE", "4"]
extended-description = "A professional vector illustration editor."
depends = "$auto"
section = "graphics"
assets = [
  ["target/release/photonic", "usr/bin/", "755"],
  ["packaging/photonic.desktop", "usr/share/applications/", "644"],
  ["assets/icons/photonic.png", "usr/share/icons/hicolor/256x256/apps/", "644"],
  ["packaging/photonic.xml", "usr/share/mime/packages/", "644"],
]
maintainer-scripts = "packaging/deb-scripts/"
```

The `photonic.xml` is a `shared-mime-info` XML file registering the `application/x-photonic`
MIME type and `.photonic` extension. Post-install scripts call `update-mime-database` and
`update-desktop-database`.

**Flatpak** (optional, lower priority): write a `dev.photonic.app.yaml` manifest using
the `org.gnome.Platform` SDK. Flatpak requires all deps to be declared; the main
challenge is wgpu/Vulkan inside the sandbox (needs `--device=dri` permission). Defer
until AppImage/.deb are validated.

### File association — cross-platform

| Platform | Mechanism | Current state |
|----------|-----------|---------------|
| Windows  | Registry `HKEY_CLASSES_ROOT\.photonic` | Runtime in `main.rs:581`; should move to installer |
| macOS    | `Info.plist` `CFBundleDocumentTypes` + UTI | Not yet; handled by `cargo-bundle` |
| Linux    | `.desktop` + `shared-mime-info` XML | Not yet; handled by deb post-install |

Once the installer handles Windows registration, the runtime `register_file_association()`
at `main.rs:92` should be guarded to run only when not installed (i.e., running from a
plain extracted binary) to avoid privilege errors.

## Affected Modules

- `crates/photonic-app/src/main.rs` — `register_file_association()` (lines 581, 636);
  add guard for installed-vs-portable detection
- `packaging/` — new directory: `photonic.desktop`, `photonic.xml` (MIME), `photonic.iss`
  or `wix/main.wxs`, `deb-scripts/postinst`
- `assets/icons/` — new directory: `photonic.icns`, `photonic.ico`, `photonic.png`
  (from `logo_improved.svg`)
- `.github/workflows/release.yml` (from issue #56) — MSI, DMG, AppImage, deb build steps
- `Cargo.toml` — `[package.metadata.wix]`, `[package.metadata.bundle]`,
  `[package.metadata.deb]` sections

## Risks & Open Questions

- **Icon assets**: only `logo_improved.svg` and `logo_current.png` exist at repo root.
  `.icns` and `.ico` multi-resolution icon files need to be generated; an `assets/icons/`
  directory does not yet exist.
- **`register_file_association()` conflict**: running it at startup as SYSTEM (via
  installer) and at user login (current code) can produce duplicate or conflicting
  registry entries on Windows. Decide on "installer owns it, runtime is a portable-mode
  fallback" and update `main.rs` accordingly.
- **macOS `.app` bundle structure**: `cargo-bundle` produces a valid bundle, but wgpu's
  Metal backend requires the Metal shader compiler at build time and `libc++` at runtime
  (both present on macOS). Verify the bundle is self-contained (no missing dylibs) with
  `otool -L`.
- **AppImage FUSE on modern distros**: AppImages require FUSE 2 or `libfuse2`, which is
  not installed by default on Ubuntu 22.04+. Either ship a Type 2 AppImage with
  `--appimage-extract-and-run` fallback, or document the `sudo apt install libfuse2` step.
- **Flatpak Vulkan sandbox**: wgpu inside a Flatpak sandbox needs `--device=dri` in the
  manifest and the Mesa ICD mounted correctly. This is non-trivial; treat Flatpak as a
  stretch goal.

## Acceptance Criteria

- [ ] Windows: running the MSI installs Photonic to Program Files, creates a Start Menu
      shortcut, registers `.photonic` association, and appears in Add/Remove Programs.
- [ ] macOS: opening the DMG shows a drag-to-Applications window; the installed `.app`
      is signed and Gatekeeper-accepted; double-clicking a `.photonic` file opens it.
- [ ] Linux: the AppImage runs without installation on Ubuntu 22.04+; the `.deb` installs
      cleanly and registers the MIME type and `.desktop` entry.
- [ ] All installers are produced and attached by the release workflow (issue #56).

## Effort Estimate

**XL** — three platforms, three distinct toolchains (WiX, cargo-bundle/create-dmg,
cargo-deb/appimagetool), plus icon asset generation and CI integration. Each platform is
M independently; the coordination overhead and debugging make the total XL.
